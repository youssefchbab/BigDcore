\section{Methodology}
Each checker module is verified independently by instantiating only the relevant parser--checker pair, injecting synthetic byte streams targeting boundary conditions, and checking alert signals in simulation. The full-pipeline testbench (\texttt{ids\_top\_tb.vhd}) then exercises the complete chain from RMII dibit injection to alert latch output.

A key optimization is the \texttt{CLK\_FREQ\_HZ} generic override:
\begin{lstlisting}[caption={Generic override in the testbench}]
	u_dut : entity work.ids_top
	generic map (CLK_FREQ_HZ => 1000)
	port map (...);
\end{lstlisting}
At 1\,kHz, the frame counter's one-second tick fires after 1000 cycles instead of 50\,000\,000, making counter behaviour observable without long simulation runs.

\section{Test Procedures}
VHDL procedures encapsulate frame injection, driving RMII signals cycle-by-cycle including preamble, SFD, header bytes, and EOF. A complete test case reduces to a few lines:
\begin{lstlisting}[caption={Injecting a land attack}]
	send_ipv4_frame(
	src_ip   => x"C0A80101",
	dst_ip   => x"C0A80101",
	protocol => x"06",
	ttl      => x"40"
	);
	wait until rising_edge(clk);
	assert alert_land_attack = '1'
	report "Land attack not detected" severity failure;
\end{lstlisting}

\section{IPv4 Checker Verification}
All twelve R1--R12 rules were verified in a single simulation run. Each frame triggers exactly one rule; no secondary alerts were observed.

\textbf{Header Validation (R1--R7):}
R1: \texttt{ip\_version=6} → \texttt{alert\_bad\_version};
R2: \texttt{ip\_ihl=3} → \texttt{alert\_bad\_ihl};
R3: \texttt{ip\_length=10} → \texttt{alert\_bad\_length};
R4: \texttt{ip\_ttl=1} → \texttt{alert\_low\_ttl};
R5: non-zero fragment offset → \texttt{alert\_fragment};
R6: \texttt{ip\_src=0.0.0.0} → \texttt{alert\_spoof\_src};
R7: \texttt{ip\_src=ip\_dst} → \texttt{alert\_land\_attack}.

\textbf{Bogon Address Rules (R8--R10):}
R8: \texttt{ip\_src=127.0.0.1} → \texttt{alert\_bogon\_loop};
R9: \texttt{ip\_src=169.254.0.1} → \texttt{alert\_bogon\_link};
R10: \texttt{ip\_src=224.0.0.1} → \texttt{alert\_bogon\_mcast}.

\textbf{Header Abuse Rules (R11--R12):}
R11: \texttt{ip\_ihl=6} → \texttt{alert\_ip\_options};
R12: \texttt{ip\_flags[2]='1'} → \texttt{alert\_reserved\_bit}.

\section{UDP Checker Verification}
All nine U1--U9 rules were verified. Each alert fired within the expected number of clock cycles from header assembly; no secondary alerts were triggered on single-violation frames.

\textbf{Forbidden Port / Structural (U1--U3):}
U1: \texttt{udp\_dst\_port=69} → \texttt{alert\_udp\_forbidden};
U2: \texttt{udp\_length=4} → \texttt{alert\_udp\_short};
U3: \texttt{udp\_length=0} → \texttt{alert\_udp\_zero}.

\textbf{Amplification / Spoofing (U4--U5):}
U4: \texttt{udp\_src\_port=53} → \texttt{alert\_udp\_amplify};
U5: \texttt{ip\_src=ip\_dst} $\wedge$ \texttt{src\_port=dst\_port} → \texttt{alert\_udp\_land} (confirmed that satisfying only one condition produces no alert).

\textbf{Size / Service Abuse (U6--U9):}
U6: \texttt{udp\_length=1480} → \texttt{alert\_udp\_oversize};
U7: \texttt{udp\_dst\_port=0} → \texttt{alert\_udp\_port\_zero};
U8: \texttt{udp\_dst\_port=1900} → \texttt{alert\_ssdp\_abuse};
U9: \texttt{udp\_dst\_port=11211} → \texttt{alert\_memcached}.
